対象：副業Webライターを主ペルソナとするフリーランス向け案件管理Webアプリ

# 1. プロジェクト概要

## 1.1 背景

クラウドワークスやランサーズなど複数のサービスを利用する副業・フリーランス層は、応募案件、返信状況、面談、納期、報酬、入金状況をそれぞれ別の場所で管理しがちである。その結果、応募先の重複、返信や納期の見落とし、未入金の放置、収益状況の把握不足が発生する。また、案件ごとに応募文を作成する負担も大きい。

## 1.2 目的

案件の発見後から応募、選考、契約、作業、納品、入金、振り返りまでを一元管理し、利用者が重要な案件や期限を迷わず把握できる状態を実現する。MVPでは管理機能を安定して提供し、v1.0以降で通知・分析、v2.0でAI機能と外部サービス連携を追加する。

# 2. ユーザーストーリー

**As a 副業Webライター,\**
I want 案件を登録して応募状況を更新できること,\
So that どの案件にいつ応募したかを把握できる.

**As a 副業Webライター,\**
I want 納期が近い案件をダッシュボードで確認できること,\
So that 締切の見落としを防げる.

**As a 副業Webライター,\**
I want 報酬と入金状況を記録できること,\
So that 未入金案件と月間売上を把握できる.

**As a 動画編集者,\**
I want 案件ごとに進捗と納期を管理できること,\
So that 複数案件を並行して進めやすくなる.

**As a デザイナー,\**
I want 案件詳細にクライアント情報とメモを保存できること,\
So that 修正内容や注意点を一か所で確認できる.

**As a オンライン秘書,\**
I want 面談日・契約日・入金予定日を管理できること,\
So that 継続案件の予定を見落としにくくなる.

**As a 全ユーザー,\**
I want 案件一覧をステータスや期限で絞り込めること,\
So that 優先して確認すべき案件を素早く見つけられる.

**As a 全ユーザー,\**
I want 月別売上を確認できること,\
So that 活動成果を振り返り次月の目標を立てられる.

# 4. 非機能要件

| 項目 | 要件 |
|----|----|
| パフォーマンス | 主要ページの初回表示を通常の通信環境で3秒以内とする。一覧・検索操作は原則2秒以内に応答する。 |
| セキュリティ | Email＋パスワード認証を採用し、将来Google OAuthを追加する。通信はHTTPSとし、ユーザーごとにデータアクセスを分離する。パスワードは平文保存しない。 |
| 可用性 | 個人開発の初期段階では月間稼働率99%を目標とする。障害時に復旧できるようDBバックアップ方針を定める。 |
| スケーラビリティ | 初期目標は同時接続100ユーザー、登録案件10万件程度まで拡張可能な構成とする。 |
| ユーザビリティ | 重要機能は2クリック以内で到達可能とし、全ログイン画面で共通のサイドバーとヘッダーを表示する。 |
| レスポンシブ | PCを主対象としつつ、スマートフォンでも案件確認・ステータス更新ができる。 |
| アクセシビリティ | 文字サイズ、コントラスト、フォームラベル、キーボード操作に配慮する。 |
| データ保護 | 他ユーザーの案件データを閲覧・更新できないようRow Level Security等を設定する。 |

# 5. 制約条件

- 予算：学習・個人開発段階では各サービスの無料枠を基本とする。

- 期間：MVPの設計・実装・テストを2週間の目標とする。

- 技術構成：Next.js（TypeScript）＋Supabase（認証・DB）＋Vercel（ホスティング）。

- 対象端末：PCブラウザを優先し、スマートフォン表示にも対応する。

- 対象ブラウザ：最新版のGoogle Chrome、Microsoft Edge、Safariを基本とする。

- 外部連携：MVPでは実装せず、v2.0でGmail・Googleカレンダー連携を検討する。

- AI機能：MVPでは実装せず、案件管理機能の安定後に追加する。

## 6.1 共通ナビゲーション

- サイドバー：ダッシュボード、案件管理、応募・選考、作業・納期、報酬・入金、実績・分析。

- ヘッダー：「案件を登録」ボタン、通知、マイページ。

- ナビゲーション名称と表示位置は全ログイン画面で統一する。

- 構造は利用者の仕事の流れ「案件→応募→作業→納品→入金→振り返り」に合わせる。

# 7. 参考デザイン

現時点では特定サービスのUIをそのまま模倣せず、以下の方針で設計する。

- ダッシュボード：カード形式で「進行中案件」「直近納期」「未入金」「月間売上」を表示する。

- 案件一覧：表形式を基本とし、ステータス、納期、報酬を一目で比較できるようにする。

- ステータス：色だけに依存せず、テキストラベルも併記する。

- 主要操作：「案件を登録」を画面右上に固定し、迷わず開始できるようにする。

- デザイン方針：落ち着いた青系を基調とし、業務管理ツールとして信頼感と視認性を重視する。
