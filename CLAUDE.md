# プロジェクト指示

このプロジェクトは `requirements.md`（AI応募案件管理システム 要件定義書）に基づいて実装する。

- 実装や設計判断に迷ったら、まず `requirements.md` を確認すること。
- 要件定義書にない仕様は独自に推測せず、ユーザーに確認するか、要件定義書の「制約条件」「非機能要件」に沿った妥当なデフォルトを選ぶこと。
- 技術構成（制約条件セクションより）: Next.js（TypeScript）＋ Supabase（認証・DB）＋ Vercel（ホスティング）。
- 実装フェーズの優先順位は「3. 機能要件」テーブルの Phase 列（MVP → v1.0 → v2.0）に従う。MVP機能から着手する。
- 「requirements.md を読んで、この要件で実装を進めて」と指示された場合は、このファイルの内容を前提に `requirements.md` 全体を読み直してから実装を開始する。
